# 대학공시 데이터 분석 툴 — 기능 명세서

> 최종 업데이트: 2026-03-27

---

## 1. 전체 구조

```
분석 페이지 (index.html)  ←── GitHub Pages 배포, 정적 HTML
관리자 페이지 (admin.html) ←── 로컬 또는 비공개 URL, GitHub API로 JSON 편집
데이터 (data/*.json)      ←── 항목별 누적 원시 데이터
설정 (calc_rules.json 등) ←── 산식·항목·매핑 정의
```

## 2. 분석 페이지 (index.html)

### 2-1. 레이아웃

```
┌─────────────┬────────────────────────────────────────┐
│  사이드바   │  필터 바                                │
│             ├────────────────────────────────────────┤
│  ▶ 순위보기 │  [뷰 콘텐츠]                            │
│  추이분석   │                                         │
│  벤치마킹   │                                         │
│  목표시뮬   │                                         │
└─────────────┴────────────────────────────────────────┘
```

- 사이드바: 뷰 전환 네비게이션
- 사이드바 접기/펼치기 가능 (localStorage에 상태 저장)

### 2-2. 필터 바

| 필터 | 동작 |
|------|------|
| 공시항목 | 선택 시 `data/{항목키}.json` + `calc_rules.json` 동적 로드 |
| 연도 | 해당 연도 데이터로 집계 |
| 지역구분 | 전국 / 비수도권 / 동남권(부산·울산·경남) / 부산 |
| 설립구분 | 전체 / 사립 |
| 대학구분 | 일반대학(대학교+산업대학) / 전체 / 교육대학 포함 |

- **추이 분석 뷰 활성 시**: 공시항목 선택만 표시, 나머지 필터 숨김
- **산포도 뷰 활성 시**: X/Y축 항목, 연도, 설립·지역 필터 표시

### 2-3. 순위 보기 뷰

**KPI 바** (우리 대학 = 부산외국어대학교 기준)
- 상위 백분율 / 전체 순위 / 지표값 3개 카드
- 순위(`_rank`)는 `rankKey` 기준 내림차순 고정 (표시 정렬과 무관)
- 폰트: 숫자 크게 / 단위(%, 위) 작게

**순위 테이블**
- 헤더 클릭으로 컬럼 정렬 (rankKey는 고정, 표시만 변경)
- 우리 대학 행 강조 표시
- 대학명 검색 필터 (실시간)
- 페이지네이션 (50행 단위, 항상 화면 하단에 고정 표시)
- CSV 내보내기 (순위 컬럼 포함)

**데이터 집계 (`DataService.aggregateByUniversity`)**
1. 연도 필터 → 캠퍼스별 raw 행 그룹핑
2. `merge_rules.json` 기준으로 캠퍼스 합산 (sum / skip / master)
3. `calc_rules.json` 산식 적용 (2단계):
   - 1단계: 일반 산식 (`numerator` / `denominator_base` / `exclude_rows`)
   - 2단계: `min_of` 산식 (1단계 결과 기반)
4. `기준대학.json`에서 지역·설립구분·대학구분 병합
5. `_rank` 필드 부여 (rankKey 기준 내림차순)

**`exclude_rows` 처리**
- 대학 단위 합산 전, 원시 행 레벨에서 필터링
- 예: `{"계열": ["의학계열"]}` → 의학계열 학과 행을 분자/분모 합산에서 제외

### 2-4. 추이 분석 뷰

```
┌──────────────────┬──────────────────────────────────┐
│ 좌측 패널        │ ECharts 꺾은선 차트               │
│ ─ 비교 그룹 ─    │ X축: 연도 / Y축: 지표값           │
│ ☑ 전국 평균      │ 우리 대학: 굵은 실선              │
│ ☑ 전국 사립      │ 그룹 평균: 점선 (각기 다른 색)    │
│ ☑ 비수도권       │ 추가 대학: 실선                   │
│ ☑ 동남권         ├──────────────────────────────────┤
│ □ 부산           │ 연도별 수치 요약 테이블            │
│ ─ 연도 선택 ─    │                                   │
│ ☑ 2020년 …       │                                   │
│ ─ Y축 범위 ─     │                                   │
│ [최소] ~ [최대]  │                                   │
│ [적용]           │                                   │
│ ─ 대학 추가 ─    │                                   │
│ [검색창]         │                                   │
└──────────────────┴──────────────────────────────────┘
```

**비교 그룹**

| 그룹 | 필터 |
|------|------|
| 전국 평균 | 기본 필터만 |
| 전국 사립 | 설립구분=사립 |
| 비수도권 | 수도권여부=N |
| 동남권 | 지역∈{부산,울산,경남} |
| 부산 | 지역=부산 |

**그룹 평균 산출 필터 (고정)**
- 설립구분: 국공립·사립만 (특별법국립·특별법법인·기타 제외)
- 대학구분: 대학교·산업대학만
- 이상치: 3σ 밖 제거

**연도 선택**: 체크박스 복수 선택 → X축 연도 필터링 (기본: 최근 5년)

**Y축 범위**: 최소·최대값 입력 후 "적용" → ECharts `yAxis.min`/`max` 반영

**대학 추가**: `field_mapping.json`의 전체 대학명 datalist로 검색, 태그로 추가·삭제

### 2-5. 순위 변동 뷰 (Bump Chart)

- 연도별 순위 변화를 선으로 연결한 Bump 차트
- 우리 대학 + 추가 대학을 강조 표시
- 필터(지역·설립·대학구분)가 순위 모집단에 영향

### 2-6. 벤치마킹 뷰

- `benchmark_cache.json` 기반 레이더 차트 / 히트맵 / 갭 분석 차트
- 우리 대학과 비교 대학(또는 그룹 평균)을 다차원 비교
- **갭 분석 차트**: xMin/xMax 동적 패딩(범위의 28%/22%)으로 데이터 레이블과 Y축 레이블 겹침 방지

### 2-7. 1:1 비교 뷰

- 두 대학의 지표를 나란히 비교하는 카드 레이아웃

### 2-8. 산포도 뷰

- `benchmarkCache` 기반 — X/Y축 항목, 연도, 설립·지역 필터는 필터 바에 위치
- X/Y축 범위(min/max) 입력은 `#scatter-view` 카드 내부 차트 위에 위치
- Auto 버튼: 해당 축 입력란을 비우고 자동 범위로 복원

### 2-9. 히트맵 뷰

- 대학 × 지표 매트릭스를 색상으로 시각화
- `benchmark_cache.json` 기반, 지역·설립구분·연도 필터 적용

### 2-10. 목표 시뮬레이터 뷰

- 특정 지표에서 목표 순위 달성에 필요한 지표값을 역산
- 현재 값 대비 개선 필요량 표시

## 3. 관리자 페이지 (admin.html)

### 3-1. 연결

- GitHub 저장소명(`owner/repo`) + Personal Access Token 입력
- 연결 성공 시 5개 탭 표시

### 3-2. 저장 방식

- 하단 "GitHub에 저장" 버튼 클릭 시 파일별 **순차** 저장
- 각 파일: `GET` (최신 SHA 조회) → `PUT` (내용 + SHA 전송)
- 병렬 PUT 금지 — GitHub race condition으로 SHA mismatch 발생

### 3-3. 기준대학 매핑 탭 (`data/기준대학.json`)

- 캠퍼스명 → 기준대학명 매핑 테이블 편집
- 지역·설립구분·대학구분·수도권여부 편집

### 3-4. 산식 관리 탭 (`calc_rules.json`)

- 카드 단위로 산식 편집
- "적용" 버튼 → 로컬 `calcData` 반영 (GitHub 미저장)
- 행 필터 섹션 (`exclude_rows`): 필드명 + 제외값 행 추가/삭제
- 산식 유형: 비율(`numerator`/`denominator`) / 합계 / MIN(`min_of`) / N년평균(`rolling_avg`) / left_join

### 3-5. 공시항목 탭 (`data/manifest.json`)

- 카드 단위로 항목 편집 (라벨 / 소스 파일 / 표시컬럼)
- "적용" 버튼 → 로컬 `manifestData[idx]` 반영 (GitHub 미저장)
- **필드명(KEY) 드롭다운**: `field_mapping.json` 키 (공시항목별 optgroup) + `calc_rules.json` 키 ("산식 (계산 지표)" 그룹)
- `collectManifest()`: DOM 파싱 없이 `manifestData` 직접 반환

### 3-6. 데이터 조회 탭

- `data/*.json` 원시 데이터 직접 조회 (manifest sources 기준 드롭다운)
- 연도 / 설립구분 / 지역 / 대학명 필터 + 페이지 단위 렌더링 (50행/페이지)
- 우리 대학 행 파란색 강조
- 1 MB 초과 파일은 git blobs API(raw) 자동 fallback

### 3-7. 캐시 관리 탭

- `visible` 지표 × manifest sources 기준으로 집계 대상 자동 구성
- 버튼 2개: **캐시 생성 및 저장** / **데이터 검증** (저장 없이 join 품질 리포트)
- 대학별·연도별 원시 집계 → calc_rules 산식 적용 → `benchmark_cache.json` 생성·저장
- 데이터 검증: V1(대학명 미매칭) / V2(join 키 불일치) / V4(필드 미존재) 섹션으로 리포트

## 4. 데이터 파일

### 4-1. `data/manifest.json`

항목별 화면 표시 설정.

```json
[
  {
    "indicator": "전임교원 확보율",
    "sources": ["대학_6-나-(1)_전임교원_확보율_학과별자료"],
    "columns": [
      { "key": "학생현황_계_학생정원(A)", "label": "학생정원" },
      { "key": "전임교원 확보율",         "label": "확보율(min)" }
    ]
  }
]
```

- `indicator`: `calc_rules.json`의 지표명 (키) — 순위 기준(rankKey)으로도 사용
- `sources`: `data/{항목키}.json` 파일명 배열 (확장자 제외) — 복수 소스 지원, 단일 소스도 배열로 표기
- `columns`: 표에 표시할 컬럼 목록 (순서대로) — `key`는 raw 필드명 또는 calc_rules 지표명

### 4-2. `calc_rules.json`

```json
{
  "전임교원 확보율_학생정원 기준(B/Cx100)": {
    "label": "전임교원 확보율(학생정원, %)",
    "visible": false,
    "exclude_rows": { "계열": ["의학계열"] },
    "numerator": ["전임교원 현황_학생정원 기준 전임교원(B)"],
    "denominator_base": "교원 법정정원_계_학생정원 기준(C)",
    "denominator_exclude": [],
    "multiply": 100,
    "used_in": ["대학_6-나-(1)_전임교원_확보율_학과별자료"]
  },
  "전임교원 확보율": {
    "label": "전임교원 확보율(%)",
    "visible": true,
    "sort_asc": false,
    "unit": "%",
    "decimal_places": 2,
    "min_of": [
      "전임교원 확보율_학생정원 기준(B/Cx100)",
      "전임교원 확보율_재학생 기준(B`/C`x100)"
    ]
  },
  "파견교환학생_5년평균": {
    "rolling_avg": "파견교환학생_계",
    "rolling_years": 5
  },
  "파견 교환학생 비율": {
    "label": "파견 교환학생 비율(%)",
    "visible": true,
    "left_join": true,
    "numerator": ["파견교환학생_5년평균"],
    "denominator_base": "재학생_계(C)",
    "multiply": 100,
    "used_in": ["대학_교환학생_파견현황"]
  }
}
```

산식 유형:
- **비율**: `numerator` + `denominator_base` + `multiply`
- **합계**: `numerator`만 사용 (denominator 없음)
- **MIN**: `min_of` — 1단계 산식 결과 중 최솟값
- **N년평균**: `rolling_avg` + `rolling_years` — 지정 필드의 최근 N년 평균 (다른 산식의 numerator에서 참조)
- **left_join**: `true`이면 sources[1](분모 소스)이 primary — 교환학생을 공시 안 한 대학도 0%로 포함

상세 내용 → `docs/architecture.md` 3-1절

### 4-3. `data/{항목키}.json`

```json
[
  {
    "기준연도": 2023,
    "공시연도": 2024,
    "기준대학명": "서울대학교",
    "학과명": "컴퓨터공학부",
    "학생현황_계_학생정원(A)": 120,
    "전임교원 현황_학생정원 기준 전임교원(B)": 15
  }
]
```

- raw 자수만 저장, 비율은 저장 안 함
- 최근 5년 보관 (5년 초과 연도 자동 제외)

### 4-4. `data/기준대학.json`

```json
[
  {
    "대학명": "연세대학교(미래캠퍼스)",
    "기준대학명": "연세대학교",
    "설립구분": "사립",
    "대학구분": "대학교",
    "지역": "강원"
  }
]
```

- `설립구분` 정규화값: `국공립` / `사립` / 특별법
- `대학구분` 값: 대학교 / 산업대학 / 교육대학 / 전문대학 등
- `수도권여부`: JSON에 저장하지 않음 — 코드에서 METRO Set으로 자동 계산
- 추이 그룹 평균에는 `국공립`+`사립`, `대학교`+`산업대학`만 포함

## 5. TREND_GROUPS 상수 (css/tokens.css 기준 색상)

| 상수명 | 색상 | 필터 조건 |
|--------|------|-----------|

| 전국 사립 | `#f59e0b` (`--trend-private`) | 설립구분=사립 |
| 비수도권 | `#22c55e` (`--trend-non-metro`) | 수도권여부=N |

우리 대학 선: `#1a56db` (`--primary`, 굵은 실선, 값 라벨 표시)
